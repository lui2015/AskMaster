# 「问大师」产品需求文档（PRD）

> 版本：v1.0
> 状态：待评审
> 更新日期：2026-07-22

---

## 一、产品概述

### 1.1 产品定位
「问大师」是一款以「投资大师人格化 AI 助手」为核心的智能问答应用。用户可以选择内置的投资大师（如巴菲特、彼得·林奇、索罗斯），围绕投资、理财、公司分析、市场判断等问题进行提问，AI 将**代入对应大师的投资理念、语言风格与思维框架**给出回答。

同时，应用支持用户**自定义编辑与新增大师**，通过预设角色指令（System Prompt）打造属于自己的「私人智囊」，并可自由切换和接入**多种国产大模型**。

### 1.2 参考对象
本产品交互与整体风格参考 `ValueMind`（深色科技风、左侧配置+会话列表、右侧对话主区、底部输入框）。在其基础上做核心能力升级：

| 对比项 | ValueMind | 问大师 |
| --- | --- | --- |
| 核心角色 | 单一（巴菲特视角） | 多大师，可切换 |
| 角色可编辑 | 否 | 支持编辑/新增/删除 |
| 模型支持 | OpenAI 兼容 | 多种国产大模型 + OpenAI 兼容 |
| 使用场景 | A 股个股分析 | 通用投资问答 + 个性化咨询 |

### 1.3 目标用户
- 个人投资者：希望获得贴近某位大师思路的投资建议与思维启发。
- 投资学习者：想理解不同投资流派（价值投资、成长投资、宏观对冲）的差异。
- 进阶玩家：想自定义专属顾问角色、接入自有大模型 API 的用户。

### 1.4 核心价值
1. **人格化**：不是冷冰冰的 AI，而是「大师附身」，回答有立场、有风格、有理念。
2. **可扩展**：大师不是写死的，用户可自建角色库。
3. **国产化**：全面支持国产大模型，合规、低延迟、成本可控。

## 二、核心需求（Core Requirements）

### 需求一：内置三位投资大师，代入理念作答

#### 2.1.1 内置大师清单

| 大师 | 投资流派 | 核心理念关键词 | 语言风格 |
| --- | --- | --- | --- |
| **沃伦·巴菲特** | 价值投资 | 护城河、安全边际、长期持有、能力圈、复利、别人贪婪我恐惧 | 睿智幽默、善用比喻、通俗易懂 |
| **彼得·林奇** | 成长投资 | 从生活中发现牛股、PEG、十倍股、投资你了解的、Tenbagger | 亲和接地气、鼓励独立思考 |
| **乔治·索罗斯** | 宏观对冲/投机 | 反身性理论、市场非理性、趋势与拐点、盛衰周期、承认错误 | 犀利深邃、强调不确定性与风险 |

#### 2.1.2 功能描述
1. 用户进入应用后，可在「大师选择区」选择一位大师开启对话。
2. 每位大师内置一段**角色指令（System Prompt）**，约束 AI 从该大师的：
   - 投资哲学与核心原则
   - 决策框架与分析维度
   - 语言风格与口头禅
   - 明确的能力边界（如：巴菲特不追热点、索罗斯强调风险）
   三个层面进行回答。
3. 对话过程中，AI 回答应**明显体现大师个人特色**，而非通用泛答。
4. 支持在同一会话中查看当前对话所属大师（头像+名称标识）。

#### 2.1.3 角色指令示例（巴菲特）
```
你是沃伦·巴菲特，全球最著名的价值投资者。请始终以巴菲特的身份、
理念和语气回答用户的投资问题。

【核心原则】
1. 只投资自己能理解的生意（能力圈原则）。
2. 寻找具有宽阔"护城河"的优秀企业。
3. 坚持"安全边际"，好价格买好公司。
4. 长期持有，让复利发挥作用；不做频繁交易。
5. 别人贪婪时恐惧，别人恐惧时贪婪。

【回答方式】
- 语言睿智、幽默、通俗，善用生活化比喻（如"滚雪球""湿雪长坡"）。
- 引导用户关注企业基本面与长期价值，而非短期股价波动。
- 遇到投机性、追热点、加杠杆的问题，应善意提醒风险。
- 不预测短期市场走势，不推荐具体买卖点。

【免责声明】
所有观点仅供学习参考，不构成投资建议。投资有风险，决策需谨慎。
```

### 需求二：大师的编辑、新增与角色指令管理

#### 2.2.1 功能描述
1. **大师列表管理**：应用提供「大师管理」入口，展示所有大师（内置 + 自定义）。
2. **新增大师**：用户可创建新的大师角色，填写以下字段：

   | 字段 | 必填 | 说明 |
   | --- | --- | --- |
   | 大师名称 | 是 | 如「查理·芒格」「张三」 |
   | 头像 | 否 | 上传图片 / 选择内置图标 / Emoji，默认自动生成 |
   | 一句话简介 | 否 | 展示在卡片副标题 |
   | 角色指令（System Prompt） | 是 | 核心，定义角色人格与作答规则 |
   | 开场白 | 否 | 用户开启会话时的第一句问候 |
   | 推荐提问（示例问题） | 否 | 引导用户快速提问，最多 4 条 |
   | 关联模型 | 否 | 可为该大师指定默认模型，不填则用全局模型 |

3. **编辑大师**：可编辑自定义大师的全部字段；内置大师支持「复制为副本后编辑」或「覆盖式编辑并可恢复默认」。
4. **删除大师**：自定义大师可删除；内置大师不可删除，仅可隐藏/恢复默认。
5. **预设角色指令引导**：新增大师时提供「模板 + 填空式引导」，降低编写 Prompt 门槛（如：性格、擅长领域、语气、禁忌）。
6. 大师数据**本地持久化**（浏览器 LocalStorage / IndexedDB），支持导入/导出为 JSON 便于备份与分享。

#### 2.2.2 交互要点
- 大师以「卡片网格」或「左侧列表」形式展示，点击即可切换当前对话对象。
- 角色指令编辑区提供实时字数统计与保存/测试按钮。

### 需求三：支持多种国产大模型

#### 2.3.1 支持模型清单（首期）

| 厂商 | 模型示例 | 接入方式 |
| --- | --- | --- |
| 深度求索 DeepSeek | deepseek-chat、deepseek-reasoner | OpenAI 兼容接口 |
| 阿里通义千问 | qwen-plus、qwen-max、qwen-turbo | OpenAI 兼容 / DashScope |
| 智谱 GLM | glm-4、glm-4-plus、glm-4-flash | 官方兼容接口 |
| 月之暗面 Kimi | moonshot-v1-8k / 32k / 128k | OpenAI 兼容接口 |
| 百度文心一言 | ernie-4.0、ernie-3.5 | 千帆平台接口 |
| 腾讯混元 | hunyuan-pro、hunyuan-standard | OpenAI 兼容接口 |
| 字节豆包 | doubao-pro、doubao-lite | 火山方舟接口 |
| （可选）OpenAI 兼容 | 任意兼容 OpenAI 协议的自定义模型 | 自定义 Base URL |

> 说明：优先统一走「OpenAI 兼容协议」适配，最大化复用同一套请求逻辑；对非标准接口（如文心、豆包）做单独适配层。

#### 2.3.2 功能描述
1. **模型配置区**（参考 ValueMind 左下角）：
   - **快速预设**：一键填充某厂商的 Base URL 与模型名（如点击「DeepSeek」自动填充 `https://api.deepseek.com`）。
   - **API Key**：密文输入，支持显示/隐藏切换，本地加密存储，不上传服务器。
   - **Base URL**：OpenAI 兼容接口地址。
   - **模型名**：如 `deepseek-chat` / `qwen-plus` / `glm-4`。
   - **高级参数**（可折叠）：temperature、max_tokens、top_p 等。
2. **测试连接**：点击后发送一条测试请求，实时反馈连通性与鉴权是否成功。
3. **保存配置**：保存后全局生效；支持配置多套并快速切换。
4. **流式输出**：AI 回答支持打字机式流式返回（SSE），提升体验。
5. **模型与大师解耦**：可为不同大师指定不同模型（如让 A 大师用 deepseek-reasoner 深度思考）。

### 需求四：接入实时行情 / 个股数据

> 已确认：本产品需接入实时行情与个股数据，让大师的回答有真实数据支撑（参考 ValueMind 的 A 股个股分析能力）。

#### 2.4.1 功能描述
1. **个股/公司识别**：用户在提问中输入股票名称或代码（如「贵州茅台 / 600519」），系统自动识别并拉取对应个股数据。
   - 支持 A 股（沪 / 深 / 北交所）证券简称与 6 位代码。
   - 无法匹配时给出友好提示与输入建议（参考截图红色错误卡片：用简称、用 6 位代码、去掉「股份/集团」后缀）。
2. **数据注入对话**：将拉取到的行情/财务数据整理后，作为上下文一并提供给大模型，使大师能**结合真实数据**给出分析，而非凭空作答。
3. **分析模式**（参考截图底部的模式选择）：
   - **个股分析**：针对单只股票，从大师视角解读基本面、估值、护城河等。
   - **同行业对比**：拉取同行业多只个股，横向对比。
4. **数据展示**：在对话区以卡片/表格形式展示关键数据（现价、涨跌幅、市盈率、市值等），再叠加大师的文字解读。
5. **免责声明**：数据仅供参考、可能存在延迟，分析不构成投资建议。

#### 2.4.2 数据范围（首期）

| 类别 | 字段示例 |
| --- | --- |
| 行情 | 现价、涨跌幅、开/高/低、成交量、成交额 |
| 估值 | 市盈率（PE）、市净率（PB）、总市值、流通市值 |
| 财务 | 营收、净利润、毛利率、ROE、资产负债率（近几期/年报） |
| 基本信息 | 所属行业、主营业务、上市日期 |

#### 2.4.3 数据源方案（建议）
- 首期接入公开行情数据接口（如新浪财经、东方财富、腾讯财经等公开行情，或第三方数据服务 API）。
- 抽象「数据源适配层」，接口可替换，便于后续切换到更稳定/合规的数据供应商。
- 做本地缓存与请求节流，避免频繁拉取被限流。

> 说明：因本期不做后端代理（纯前端），需评估目标数据接口是否存在跨域（CORS）限制；若受限，可考虑使用公开的 JSONP 行情接口，或后续按需再引入轻量代理（列入 P2 备选）。

#### 2.4.4 与大师能力的边界
- 大师负责「解读与判断」，数据接口负责「提供事实」；两者分工明确。
- 若个股数据缺失，仍可进行**通用投资问答**，不阻断对话。
- 港股、美股、基金首期不支持（与截图一致），后续按需扩展。

### 3.2 关键页面
1. **主对话页**：大师切换 + 会话管理 + 对话交互 + 模型配置。
2. **大师管理页/弹窗**：大师卡片列表、新增/编辑表单、导入导出。
3. **设置页**：全局模型配置、主题、数据管理（清空/备份）。

### 3.3 交互细节
- 切换大师时提示「是否新建会话」，避免不同大师混在一个上下文中。
- 消息支持复制、重新生成、删除。
- 网络异常、鉴权失败、余额不足等给出友好错误提示（参考截图中的红色错误卡片样式）。
- 输入个股时提供「分析模式」切换（个股分析 / 同行业对比）与识别失败的引导提示。
- 拉取到的行情数据以数据卡片/表格展示，叠加大师文字解读。
- 每条回答底部固定展示免责声明（含数据仅供参考、可能延迟）。

## 四、非功能性需求

| 类别 | 要求 |
| --- | --- |
| 性能 | 首屏加载 < 2s；流式响应首字延迟 < 2s |
| 安全 | API Key 仅存本地并加密；请求不经过自建服务器（纯前端）或走安全代理 |
| 兼容性 | 支持主流现代浏览器（Chrome / Edge / Safari）；响应式适配移动端 |
| 隐私 | 对话数据默认本地存储，不上传；提供一键清空 |
| 可用性 | 断网/接口报错有明确提示与重试入口 |
| 可维护性 | 模型适配层可插拔，新增模型改动最小 |

## 五、技术方案建议

### 5.1 技术选型（建议）
- **前端**：Vue 3 / React + TypeScript + Vite。
- **UI**：TailwindCSS（深色主题）+ 组件库（如 Naive UI / shadcn）。
- **状态管理**：Pinia / Zustand。
- **数据存储**：IndexedDB（对话历史）+ LocalStorage（配置/大师）。

- **部署形态**：纯前端静态应用，可托管于任意静态服务器；如需保护 Key，可选配轻量后端代理。

## 七、风险与合规

1. **投资建议合规**：所有回答**必须附免责声明**，明确"仅供学习参考，不构成投资建议"，避免误导。
2. **大师肖像与姓名**：使用真实人物姓名需注意合规，建议标注"AI 模拟，非本人观点"。
3. **API Key 安全**：纯前端存储需明确告知用户风险，敏感场景建议使用后端代理。
4. **模型合规**：优先接入已备案的国产大模型，符合国内监管要求。
5. **内容安全**：接入内容审核，过滤违规提问与输出。

## 八、需求确认结论（Confirmed）

| # | 问题 | 结论 | 对方案的影响 |
| --- | --- | --- | --- |
| 1 | 是否需要账号体系与云端同步 | **暂不需要**，纯本地存储 | 数据全部存本地（IndexedDB/LocalStorage），无登录、无后端账号 |
| 2 | 是否接入实时行情/个股数据 | **需要** | 新增「需求四」，接入 A 股行情/个股数据并注入对话；支持个股分析与同行业对比 |
| 3 | 是否需要后端代理保护 API Key | **暂不需要**，纯前端方案 | Key 仅本地加密存储；需注意行情接口的跨域限制，必要时后续再评估轻量代理 |
